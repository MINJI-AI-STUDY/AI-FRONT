# Shared Workspace 레이아웃

## 공식 기준
- **채널이 teacher/student 학습의 공식 메인 단위다.**
- PDF, AI 질의응답, 채널 대화, 문제 생성/검토/배포, 학생 풀이 진입은 모두 채널 문맥에서 이어진다.
- distribution code는 보조 참여 수단이며, 공식 메인 내비게이션 단위는 아니다.

## 공통 구조
- 상단 강의 플랫폼형 nav
- 좌측 floating rail
- 좌측 채널 목록 패널(탐색 전용, teacher/student 공통 기본 노출)
- 중앙 문서/PDF canvas
- 우측 역할별 작업 패널
- desktop 기준 teacher/student 모두 `좌측 탐색 / 중앙 PDF / 우측 보조작업` 3분할을 유지한다.
- desktop에서는 teacher 채널 워크스페이스의 전체 패널 collapse UX를 두지 않는다.

## teacher 워크스페이스
- 중앙: 채널에 귀속된 PDF inline viewer
- 우측: 채널 관리, 현재 PDF 선택, 채널 최근 메시지, 문제 생성 설정, 최근 생성 세트 검토, 배포 상태/코드 확인
- 교사의 공식 출제 흐름은 `teacher channel workspace`에서 시작한다.
- 좌측 channel list는 탐색만 담당하고, 운영 액션은 우측 보조 패널에서 이어진다.

## student 워크스페이스
- 중앙: 채널에 귀속된 PDF inline viewer
- 우측: `AI 도우미` / `채널 대화` 전환형 패널
- 학생의 공식 학습 흐름은 `student channel workspace`에서 시작한다.
- 문제 풀이 진입도 활성 question set이 있으면 채널 문맥에서 바로 노출된다.
- AI 답변은 `핵심 답변 → 필요 시 전체 답변/근거 펼치기` 순서를 기본으로 한다.
- 오답 후속 학습은 결과 확인 화면/모달에서 문항별 `AI 해설 요청`으로 채널 문맥에 다시 연결한다.

## 모달 사용 원칙
- teacher: 생성 결과 JSON 확대
- student: 제출 직후 결과 확인은 modal-first, 직접 진입/새로고침 대비용 결과 페이지 fallback 유지
- 단, 메인 학습 맥락(PDF/AI/chat/question entry)은 채널 워크스페이스 안에서 끊기지 않아야 한다.
