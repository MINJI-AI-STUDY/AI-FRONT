#SM|# Shared Workspace 레이아웃
#KM|
#ST|## 공식 기준
#ZV|- **채널이 teacher/student 학습의 공식 메인 단위다.**
#WB|- PDF, AI 질의응답, 채널 대화, 문제 생성/검토/배포, 학생 풀이 진입은 모두 채널 문맥에서 이어진다.
#MY|- distribution code는 보조 참여 수단이며, 공식 메인 내비게이션 단위는 아니다.
#HN|
#VW|## 공통 구조
#QJ|- 문서 우선 shell
#TP|- 좌측 내비게이션은 on-demand overlay로 열고 닫는다.
#WR|- 좌측 채널 목록 패널은 기본 노출이 아니라 필요 시 노출되는 탐색 전용 레이어다.
#QB|- 중앙 문서/PDF canvas가 항상 주 작업면이다.
#TH|- 우측 역할별 작업 패널은 기본 닫힘이며 필요 시 보조 패널/드로어로 연다.
#SJ|- tablet-first 기준으로 `중앙 PDF 중심 / 좌측 오버레이 nav / 우측 보조 패널` 구조를 유지한다.
#YY|- desktop에서도 고정된 persistent side-column layout이 아니라, 문서 우선 shell의 확장 형태로 동작한다.
#VP|
#RM|## teacher 워크스페이스
#SX|- 중앙: 채널에 귀속된 PDF inline viewer(문서 우선)
#KJ|- 우측: 기본 닫힌 보조 패널에서 채널 관리, 현재 PDF 선택, 채널 최근 메시지, 문제 생성 설정, 최근 생성 세트 검토, 배포 상태/코드 확인
#NB|- 교사의 공식 출제 흐름은 `teacher channel workspace`에서 시작한다.
#SM|- 좌측 channel list는 필요 시 오버레이로 열어 탐색만 담당하고, 운영 액션은 우측 보조 패널에서 이어진다.
#LQ|- 교사/학생 공통 shell 철학은 같고, 역할 차이는 우측 패널의 내용과 기본 진입점에만 있다.
#ZP|
#MP|## student 워크스페이스
#SX|- 중앙: 채널에 귀속된 PDF inline viewer(문서 우선)
#MB|- 우측: 기본 닫힌 `AI 도우미` / `채널 대화` 전환형 패널
#TQ|- 학생의 공식 학습 흐름은 `student channel workspace`에서 시작한다.
#ZJ|- 문제 풀이 진입도 활성 question set이 있으면 채널 문맥에서 바로 노출된다.
#WW|- AI 답변은 `핵심 답변 → 필요 시 전체 답변/근거 펼치기` 순서를 기본으로 한다.
#YY|- 오답 후속 학습은 결과 확인 화면/모달에서 문항별 `AI 해설 요청`으로 채널 문맥에 다시 연결한다.
#LG|- 좌측 내비게이션은 오버레이로 필요할 때만 열고, 중앙 PDF를 가리지 않는 것이 기본이다.
#ZM|
#PS|## 모달 사용 원칙
#BQ|- teacher: 생성 결과 JSON 확대
#YB|- student: 제출 직후 결과 확인은 modal-first, 직접 진입/새로고침 대비용 결과 페이지 fallback 유지
#TR|- 단, 메인 학습 맥락(PDF/AI/chat/question entry)은 채널 워크스페이스 안에서 끊기지 않아야 한다.
#PN|- PDF viewer는 현재 iframe/native 기반이며, 두 쪽 동시 제어 같은 보장은 하지 않고 페이지 모드/단일 페이지 중심의 honest affordance만 제공한다.
