#HQ|# Teacher Workspace 상세 화면 명세
#KM|
#SY|## 화면 경로
#KQ|- 공식 메인: `/teacher/channels/:channelId/workspace`
#MP|- 보조 생성/검토 화면: `/teacher/materials/:materialId/generate`, `/teacher/question-sets/:questionSetId/review` (호환 유지용)
#SY|
#PH|## 목적
#RQ|- 교사가 채널에 귀속된 PDF를 보면서 문제 생성, 최근 생성 세트 검토, 배포 상태 확인을 수행한다.
#QQ|- 교사의 공식 출제 단위는 채널이며, material/question set은 채널 문맥 안에서 다뤄진다.
#SK|
#VB|## 주요 영역
#RT|| 영역 | 설명 |
#NJ|| --- | --- |
#VY|| 상단 header | 채널 제목, 역할 메타, 교사 홈 복귀 |
#QT|| 좌측 channel list | 필요 시 여는 오버레이 탐색 패널 |
#JB|| 중앙 문서 canvas | 현재 채널에 연결된 PDF inline viewer, 화면의 주 작업면 |
#BK|| 우측 보조작업 패널 | 기본 닫힌 보조 패널, 채널 관리, 현재 PDF 선택, 채널 최근 메시지, 문제 생성/검토 모달 진입 |
#NS|| 문제 생성 모달 | 현재 채널+선택 PDF 기준 생성 파라미터 입력 후 생성 |
#BN|| 문제 검토/배포 모달 | 문항 제외 토글, 마감 설정, 최종 확인 후 즉시 배포 |
#YQ|
#XV|## 상태
#PZ|| 상태 | 설명 |
#XK|| --- | --- |
#BW|| loading | 자료 로딩 중 |
#WQ|| error | 자료 조회 실패 |
#VP|| idle | 생성 전 |
#XY|| generating | 생성 요청 중 |
#NZ|| generated | JSON 결과 표시 |
#JZ|| published | 배포 코드 / 마감 상태 확인 |
#ZM|
#SY|## 액션
#QT|- 문제 생성 모달 열기
#TY|- 최근 검토 세트 모달 열기
#JM|- 검토 모달에서 최종 확인 후 즉시 배포
#BB|- 우측 패널에서 채널 전체 최근 메시지 확인
#ZS|- 우측 패널에서 현재 PDF 전환 및 채널 이름 수정
#RB|
#WV|## UX 원칙
#TT|- PDF가 항상 중앙 주인공이어야 한다.
#WS|- 문제 생성/검토/배포는 채널 문맥에서 시작해야 한다.
#KP|- 생성→검토→배포 기본 흐름은 채널 워크스페이스 모달 안에서 끝낸다.
#QR|- material standalone 페이지는 보조/레거시 작업 화면이며, 공식 메인 학습/출제 단위는 채널이다.
#YT|- desktop에서도 always-open side column이 아니라, 문서 우선 shell의 확장 형태로 본다.
#VZ|- 좌측은 오버레이 탐색, 우측은 기본 닫힌 운영/보조작업 패널로 역할을 분리한다.
#LF|- 현재 PDF viewer는 iframe/native 기반이므로, 진짜 양면 제어를 약속하지 않고 페이지 중심의 honest affordance만 노출한다.
