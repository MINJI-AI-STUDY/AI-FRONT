# Teacher Workspace 상세 화면 명세

## 화면 경로
- 공식 메인: `/teacher/channels/:channelId/workspace`
- 보조 생성/검토 화면: `/teacher/materials/:materialId/generate`, `/teacher/question-sets/:questionSetId/review` (호환 유지용)

## 목적
- 교사가 채널에 귀속된 PDF를 보면서 문제 생성, 최근 생성 세트 검토, 배포 상태 확인을 수행한다.
- 교사의 공식 출제 단위는 채널이며, material/question set은 채널 문맥 안에서 다뤄진다.

## 주요 영역
| 영역 | 설명 |
| --- | --- |
| 상단 header | 채널 제목, 역할 메타, 교사 홈 복귀, compact one-line shell |
| 좌측 channel list | 필요 시 여는 overlay drawer형 탐색 패널 |
| 중앙 문서 canvas | 현재 채널에 연결된 PDF inline viewer, first paint의 주 작업면 |
| 우측 보조작업 패널 | 기본 닫힌 contextual sheet/drawer, 최근 배포 세트 대시보드 요약, 채널 관리, 현재 PDF 선택, 채널 최근 메시지, 문제 생성/검토 모달 진입 |
| 문제 생성 모달 | 현재 채널+선택 PDF 기준 생성 파라미터 입력 후 생성 |
| 문제 검토/배포 모달 | 문항 제외 토글, 마감 설정, 최종 확인 후 즉시 배포 |

## 상태
| 상태 | 설명 |
| --- | --- |
| loading | 자료 로딩 중 |
| error | 자료 조회 실패 |
| idle | 생성 전 |
| generating | 생성 요청 중 |
| generated | JSON 결과 표시 |
| published | 배포 코드 / 마감 상태 확인 |

## 액션
- 문제 생성 모달 열기
- 최근 검토 세트 모달 열기
- 검토 모달에서 최종 확인 후 즉시 배포
- 우측 패널에서 채널 전체 최근 메시지 확인
- 우측 패널에서 현재 PDF 전환 및 채널 이름 수정

## UX 원칙
- PDF가 항상 중앙 주인공이어야 한다.
- 문제 생성/검토/배포는 채널 문맥에서 시작해야 한다.
- 생성→검토→배포 기본 흐름은 채널 워크스페이스 모달 안에서 끝낸다.
- 최근 배포 세트의 제출 학생 수, 평균 점수, 위험 문항은 우측 패널에서 바로 읽혀야 한다.
- material standalone 페이지는 보조/레거시 작업 화면이며, 공식 메인 출제 단위는 채널이다.
- desktop에서도 always-open side column이 아니라, PDF-first shell의 확장 형태로 본다.
- 좌측은 drawer 탐색, 우측은 기본 닫힌 contextual tools overlay 패널로 역할을 분리한다.
- tablet/mobile first load에서 PDF가 먼저 읽히고, 교사 작업 CTA는 문서 흐름을 해치지 않는 범위에서 compact header와 contextual tools로 분산된다.
- 현재 PDF viewer는 iframe/native 기반이므로, 실제 양면 동시 제어를 약속하지 않고 단일 페이지 네비게이션만 제공한다.
