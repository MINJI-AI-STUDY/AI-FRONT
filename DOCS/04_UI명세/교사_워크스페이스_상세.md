#BS|#PH|## 목적
#XZ|#RQ|- 교사가 채널에 귀속된 PDF를 보면서 문제 생성, 최근 생성 세트 검토, 배포 상태 확인을 수행한다.
#QJ|#QQ|- 교사의 공식 출제 단위는 채널이며, material/question set은 채널 문맥 안에서 다뤄진다.

#ZV|#WV|## UX 원칙
#KJ|#TT|- PDF가 항상 중앙 주인공이어야 한다.
#QN|#WS|- 문제 생성/검토/배포는 채널 문맥에서 시작해야 한다.
#XK|#KP|- 생성→검토→배포 기본 흐름은 채널 워크스페이스 모달 안에서 끝낸다.
#PB|#QR|- material standalone 페이지는 보조/레거시 작업 화면이며, 공식 메인 학습/출제 단위는 채널이다.
#XB|#YT|- desktop에서도 always-open side column이 아니라, document-first shell의 확장 형태로 본다.
#WT|#VZ|- 좌측은 drawer 탐색, 우측은 기본 닫힌 contextual tools 패널로 역할을 분리한다.
#LM|#QX|- tablet/mobile first load에서 PDF가 먼저 읽히고, 교사 작업 CTA는 문서 흐름을 해치지 않는 범위에서 compact header와 contextual tools로 분산된다.
#WH|#LF|- 현재 PDF viewer는 iframe/native 기반이므로, 진짜 양면 제어를 약속하지 않고 페이지 중심의 honest affordance만 노출한다.
