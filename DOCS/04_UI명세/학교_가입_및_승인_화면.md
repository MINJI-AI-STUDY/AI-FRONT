# 학교 가입 및 승인 화면

## 화면
- `/signup/teacher`: 교직원 가입 요청
- `/signup/student`: 학생 가입 요청
- `/operator/signup-requests`: 학교 운영자 가입 승인

## 교직원 가입 요청
- 학교 선택
- 이름
- 로그인 ID
- 비밀번호
- 학교 이메일
- 이용약관/개인정보/학생·학교 고지 동의

## 학생 가입 요청
- 학교 선택
- 학생 코드 (학교에서 발급한 고유 식별자)
- 실명 (표시용)
- PIN 입력
- 가입 요청 접수 후 승인 대기 안내 문구 표시
  - "학교 운영자 승인 후 학교·학생 코드·PIN으로 로그인할 수 있습니다"
- 이용약관/개인정보/학생 서비스 고지 동의
## 운영자 승인 화면
- 학교 선택
- 선택 학교와 최근 처리 결과를 카드 상단에서 바로 확인
- 대기 중인 가입 요청 목록
- 승인/반려 버튼
- 반려 시 사유 입력 가능
- 교직원 승인 시 provisioned 로그인 정보 확인 가능
- 학생 승인 시 학교·학생 코드·PIN 기반 로그인 안낸 제공
  - 승인 완료 메시지: "학생은 학교, 학생 코드, PIN으로 로그인합니다"
  - 요청 목록에서 학생 코드/요청 ID를 표시하여 동명이인과 계정 요청을 구분

## 운영자 범위
- 이번 MVP에서 운영자 화면은 학교 단위 가입 승인과 학교 요약 대시보드에 집중합니다.
- 학교/학급/사용자 전반 CRUD는 운영 화면 기본 범위에 포함하지 않습니다.
