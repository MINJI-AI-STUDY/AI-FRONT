#ZV|#WV|## UX 원칙
#ZW|#BZ|- 문서 맥락을 끊지 않는다.
#XK|#JS|- 문제풀이와 QA는 같은 자료 컨텍스트를 공유한다.
#JW|#PS|- AI와 채널 대화는 우측 기본 닫힘 패널 안에서 전환되며, 중앙 PDF가 항상 주인공이어야 한다.
#NS|#MB|- 활성 문제 세트 조회는 material 단독이 아니라 channel 우선 컨텍스트를 사용한다.
#WB|#YT|- distribution code는 진입 수단일 뿐, 학생의 기본 학습 단위는 채널이다.
#MC|#QA|- tablet/mobile first load에서 먼저 보여야 하는 것은 PDF와 현재 문제의 핵심 정보이며, solve/submit 같은 primary action은 main flow 안에서 바로 보이도록 한다.
#PP|#BN|- AI 답변은 `핵심 답변`을 먼저 보여주고, 전체 답변/근거는 접어서 확장한다.
#SS|#BW|- 근거 부족 계열 안내는 거절형 문구가 아니라 보수적 답변 안내로 표현한다.
#NT|#ZT|- 결과 확인에서는 오답 문항마다 `AI 해설 요청`으로 다시 채널 학습 문맥에 연결한다.
#FD|#QH|- student 질문 workspace는 solve/submit이 숨은 보조 버튼이 아니라 tablet/mobile에서도 main flow에 드러나야 한다.
#YT|#LH|- 현재 PDF viewer는 iframe/native 기반이며, 듀얼페이지 보장은 하지 않고 페이지 모드/단일 페이지 중심의 honest affordance만 제공한다.
