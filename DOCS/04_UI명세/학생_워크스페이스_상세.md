# Student Workspace 상세 화면 명세

## 화면 경로
- 공식 메인: `/student/channels/:channelId/workspace`
- 집중 풀이: `/student/question-sets/:distributionCode/workspace`

## 목적
- 학생이 같은 PDF를 보면서 채널 대화와 자료 기반 AI 질의응답을 오가고,
- 활성 문제 세트가 있으면 같은 채널 문맥에서 바로 문제 풀이로 진입한다.

## 주요 영역
| 영역 | 설명 |
| --- | --- |
| 상단 header | 채널 제목, 역할 메타, 학생 홈 복귀, compact one-line shell |
| 좌측 channel list | 필요 시 여는 overlay drawer형 탐색 패널 |
| 중앙 문서 canvas | 현재 채널에 연결된 PDF inline viewer, first paint의 주 작업면 |
| 우측 mode panel | 기본 닫힌 `AI 도우미` / `채널 대화` 전환형 contextual overlay 패널 |
| 문제 풀이 entry | 활성 question set이 있으면 채널 문맥에서 즉시 진입, tablet/mobile에서는 solve/submit 주요 액션을 main flow 안에 유지 |
| 우측 패널 모드 | `AI 도우미` / `채널 대화`를 같은 패널에서 전환 |
| 결과 확인 modal | 제출 직후 modal-first로 열리며, loading/error/empty/success 상태를 모두 가진다 |

## 상태
| 상태 | 설명 |
| --- | --- |
| loading | 문제 세트 로딩 |
| error | 문제 세트 또는 QA 요청 실패 |
| noActiveSet | 현재 채널에 연결된 배포 문제 없음 |
| answering | QA 요청 중 |
| submitting | 제출 중 |
| responded | QA 응답 표시 |
| resultModalOpen | 제출 직후 결과 modal 표시 |

## UX 원칙
- 문서 맥락을 끊지 않는다.
- 문제풀이와 QA는 같은 자료 컨텍스트를 공유한다.
- AI와 채널 대화는 우측 기본 닫힘 패널 안에서 전환되며, 중앙 PDF가 항상 주인공이어야 한다.
- 활성 문제 세트 조회는 material 단독이 아니라 channel 우선 컨텍스트를 사용한다.
- distribution code는 진입 수단일 뿐, 학생의 기본 학습 단위는 채널이다.
- tablet/mobile first load에서 먼저 보여야 하는 것은 PDF와 현재 문제의 핵심 정보이며, solve/submit 같은 primary action은 main flow 안에서 바로 보이도록 한다.
- AI 답변은 `핵심 답변`을 먼저 보여주고, 전체 답변/근거는 접어서 확장한다.
- 근거 부족 계열 안내는 거절형 문구가 아니라 보수적 답변 안내로 표현한다.
- 결과 확인에서는 오답 문항마다 `AI 해설 요청`으로 다시 채널 학습 문맥에 연결한다.
- student 질문 workspace는 solve/submit이 숨은 보조 버튼이 아니라 tablet/mobile에서도 main flow에 드러나야 한다.
- readable viewport에서는 two-page spread를 기본으로 보이고, 좁은 화면/가독성 저하 시 one-page fallback으로 전환한다.
- 현재 PDF viewer는 iframe/native 기반이며, 듀얼페이지 보장은 하지 않고 page/spread 중심의 honest affordance만 제공한다.
